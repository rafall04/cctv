# backend module map

## Entry Points

- `backend/controllers/viewerController.js`: live viewer start/stop, heartbeats, runtime signals.
- `backend/controllers/playbackViewerController.js`: playback viewer start/stop, heartbeats, history, analytics.
- `backend/services/viewerSessionService.js`: live session persistence and analytics.
- `backend/services/playbackViewerSessionService.js`: playback session persistence and analytics.
- `backend/services/networkIdentityService.js`: IP to ASN/ISP resolution.
- `backend/services/networkAccessPolicyService.js`: ASN allowlist/denylist enforcement.
- `backend/routes/networkAccessPolicyRoutes.js`: authenticated admin policy CRUD endpoints.
- `backend/scripts/updateGeoLite2Asn.js`: operator script to refresh the local GeoLite2 ASN mmdb.

## ASN/ISP Flow

- Request IP is normalized from proxy headers in the session services.
- `networkIdentityService.resolveIpIdentity()` resolves ASN number and ISP/org name from the local GeoLite2 ASN database.
- `networkAccessPolicyService.enforceAccess()` evaluates the configured policy for live or playback access.
- Admin settings save rows through `/api/network-access-policies` for global, area, or camera scope.
- Successful sessions persist ASN fields in live and playback session tables and history/archive tables.
- Denied sessions return HTTP 403 from the controllers and are written to security logs.

## Persistence

- `viewer_sessions` and `viewer_session_history` store live session identity metadata.
- `playback_viewer_sessions` and `playback_viewer_session_history` store playback session identity metadata.
- `asn_access_policies` stores global/area/camera rules per access flow.
- GeoLite2 ASN updates are operator-controlled through `npm run update:geolite2-asn`; request-time code never downloads remote data.

## Verification Targets

- Controller route tests for live/playback 403 handling.
- Service tests for network identity and ASN policy enforcement.
- Migration tests or migration run against the local SQLite database.
