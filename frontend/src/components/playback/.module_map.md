# frontend/src/components/playback/.module_map.md

## Components

- `PlaybackHeader.jsx`: camera selector, playback route header, and public share trigger.
- `PlaybackVideo.jsx`: video element shell, media controls, buffering/seeking/error/empty states.
- `PlaybackTimeline.jsx`: visual segment timeline and timeline click handling surface.
- `PlaybackSegmentList.jsx`: selectable recording segment list.
- `PlaybackUsageGuide.jsx`: static usage guidance with public preview policy copy.
- `PlaybackQuickActions.jsx`: compact public playback navigation to live CCTV, area, home, and share.

## Boundaries

- Keep media lifecycle, URL state, segment loading, and playback viewer tracking in `../../pages/Playback.jsx`, `../../hooks/playback/*`, and `../../utils/playbackUrlState.js`.
- Keep these components presentational; callbacks and policy values should be passed as props.
- Public playback must not expose admin-only wording or scope. Admin playback should hide public preview limitations.
- Quick actions should stay lightweight and route-only; do not embed landing discovery, map, or area lists into playback.

## Verification

- Focused component tests: `npm test -- src/components/playback/PlaybackVideo.test.jsx src/components/playback/PlaybackUsageGuide.test.jsx`.
- Playback integration tests: `npm test -- src/pages/Playback.test.jsx src/utils/playbackSegmentSelection.test.js src/utils/publicShareUrl.test.js`.
