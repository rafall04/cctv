# MultiView Module Map

## Ownership

- `MultiViewLayout.jsx`: fullscreen multi-view shell, body scroll lock, Escape/fullscreen handling, tile layout, and per-tile status isolation.
- `MultiViewVideoItem.jsx`: one HLS camera tile inside multi-view. Owns HLS instance lifecycle, playback timeout state, viewer session lifecycle, snapshot, zoom, fullscreen, and per-tile retry UI.
- `VideoPopup.jsx`: single-camera public popup. More complete popup player supporting HLS plus external FLV/MJPEG/embed paths and header viewer stats; skips manual viewer session tracking for proxied HLS because the HLS proxy owns those sessions.
- `MultiViewButton.jsx`: floating launcher shown when cameras are selected.
- `ZoomableVideo.jsx`: shared video zoom/pan wrapper for HLS/FLV video elements.

## Critical Flow

- Landing camera selection uses `useLandingInteractions` and caps multi-view selection through `getMaxConcurrentStreams()`.
- `MultiViewLayout` renders up to three `MultiViewVideoItem` tiles and staggers startup by tile index.
- `MultiViewVideoItem` uses `streamInitQueue` for every device tier so multi-view startup does not burst all HLS sources at once.
- Internal HLS tiles may see temporary 404 manifest/level errors while MediaMTX warms a source; retry these before surfacing an error.
- Keep viewer sessions guarded against async unmount races: if a delayed `startSession` resolves after unmount, immediately stop that session; do not start a frontend session for internal/proxied HLS streams.

## Stability Rules

- Do not place transient `loadingStage` or `autoRetryCount` state in the main HLS effect dependency list; use refs for event handlers.
- Keep HLS cleanup idempotent and scoped per tile so one camera failure does not destroy another camera.
- Multi-view is HLS-only. Non-HLS external formats should remain blocked by capability checks before selection.

## Verification

- Focused multi-view stability: `npm test -- streamInitQueue.test.js MultiViewVideoItem.stability.test.js`
- Broader media popup check: `npm test -- VideoPopup.test.jsx`
- Frontend gate for release: `npm test && npm run build && npm run lint`
