# frontend/src/components/admin/settings

## Active Panels

- `GeneralSettingsPanel.jsx`: public landing/event/announcement settings used by `UnifiedSettings`.
- `AdsSettingsPanel.jsx`: public ad slot configuration and script validation.
- `BrandingSettingsPanel.jsx`: public branding settings.
- `SaweriaSettingsPanel.jsx`: donation/support settings.
- `TelegramSettingsPanel.jsx`: Telegram bot settings, multi-target monitoring groups, and notification routing rules.
- `StreamHealthSettingsPanel.jsx`: stream health policy settings.
- `PlaybackSettingsPanel.jsx`: recording/playback policy settings.
- `NetworkAccessPolicyPanel.jsx`: ASN allowlist/denylist customization for live and playback access.

## Legacy Tabs Kept For Unified Settings

- `ApiKeySettings.jsx`: placeholder API key tab.
- `BackupSettingsTab.jsx`: backup export/import tab.
- `TimezoneSettingsTab.jsx`: timezone selector tab.
- `BrandingSettingsTab.jsx`, `SaweriaSettingsTab.jsx`, `TelegramSettings.jsx`: compatibility tabs that delegate to standalone pages.

## Legacy Re-Exports

- `../../settings/*` should remain thin compatibility re-exports only.
- New settings imports should use `components/admin/settings/*`.

## Verification

- Focused tests: `npm test -- UnifiedSettings.test.jsx GeneralSettingsPanel.test.jsx AdsSettingsPanel.test.jsx`.
- Build check: `npm run build` after changing lazy/import paths or panel exports.
