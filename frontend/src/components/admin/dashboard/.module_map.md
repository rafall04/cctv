# frontend/src/components/admin/dashboard

## Components

- `DashboardSkeletons.jsx`: initial route loading skeletons for header, stats, stream table, and activity sidebar.
- `DashboardStreams.jsx`: stream priority ranking, stream table, stream drawer, and viewer-session modal.
- `DashboardSummaryCards.jsx`: top summary stat cards and attention notices.
- `DashboardSidebar.jsx`: top cameras, recent activity, and static system health indicators.

## Flow

- `pages/Dashboard.jsx` owns data loading via `useDashboardData`, navigation, modal state, and drawer state.
- Presentational modules receive data/callback props and should not fetch dashboard data directly.
- Stream ordering remains centralized in `rankDashboardStreams()` so the table and drawer stay consistent.

## Verification

- Focused frontend gate: `npm test -- Dashboard.test.jsx` when present, otherwise `npm run lint && npm run build`.
- Full frontend gate before push: `npm test && npm run build && npm run lint`.
