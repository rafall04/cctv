# frontend/src/hooks/admin/.module_map.md

## Hooks

- `useAreaFormState.js`: Area create/edit modal state, form defaults, field changes, and duplicate-name validation.
- `useDashboardData.js`: Dashboard data loading and refresh state.
- `useRecordingDashboardData.js`: Recording dashboard data loading, quick edits, refresh, and restart log state.
- `useAdminReconnectRefresh.js`: Admin reconnect refresh behavior.
- `useCameraManagementPage.js`: Camera Management data/form/filter state, camera mutations, source lifecycle result handling, and manual stream refresh action state.
- `usePlaybackTokenManagementPage.js`: Playback token list/create/edit/share/audit state, stable repeat-share messaging, and per-camera entitlement rule payload shaping.

## Boundaries

- Keep service mutations in page shells unless a hook owns the full data lifecycle. Camera Management owns its lifecycle mutations in `useCameraManagementPage.js`.
- Keep pure payload shaping in `../../utils/admin/*`.
- Keep presentation in `../../components/admin/*`.

## Verification

- Area form hook: `npm test -- src/hooks/admin/useAreaFormState.test.jsx src/pages/AreaManagement.test.jsx`.
- Dashboard hooks: `npm test -- src/hooks/admin/useRecordingDashboardData.test.jsx src/pages/Dashboard.test.jsx`.
- Camera Management hook path: `npm test -- CameraManagement.test.jsx streamRevision.test.js`.
- Playback token hook: `npm test -- src/hooks/admin/usePlaybackTokenManagementPage.test.jsx`.
