# frontend/src/hooks/playback/.module_map.md

## Hooks

- `usePlaybackSegments.js`: Loads segment lists, applies access policy defaults, chooses initial/URL-selected segments, and protects stale responses.
- `usePlaybackMediaSource.js`: Owns media source attachment, cleanup, and playback source reset behavior.
- `usePlaybackSelectionActions.js`: Handles camera/segment/timeline selection state transitions.
- `usePlaybackShareAndSnapshot.js`: Owns share URL and snapshot side effects.
- `usePlaybackViewerTracking.js`: Starts/stops playback viewer sessions after real playback starts.
- `usePlaybackTokenAccess.js`: Activates public token/share-key cookies, exposes `allowedCameraIds` and `cameraRules`, and heartbeats token sessions.

## Boundaries

- Keep pure access-scope and policy defaults in `../../utils/playbackAccessPolicy.js`.
- Keep route shell composition in `../../pages/Playback.jsx`.
- Keep presentational playback UI in `../../components/playback/*`.
- Selected-token metadata should filter public playback cameras using `allowed_camera_ids`; preserve `cam` while removing `share`/`token` URL params after activation.

- Segment/access policy changes: `npm test -- src/utils/playbackAccessPolicy.test.js src/hooks/playback/usePlaybackSegments.test.jsx src/pages/Playback.test.jsx`.
- Media lifecycle changes: `npm test -- src/hooks/playback/usePlaybackMediaSource.test.jsx src/components/playback/PlaybackVideo.test.jsx src/pages/Playback.test.jsx`.
