# frontend/src/hooks/.module_map.md

## Public Hooks

- `public/useLandingInteractions.js`: owns public popup state, URL `camera` sync, multi-view selection, and popup close behavior.
- `public/useLandingModeState.js`: owns public layout/view mode URL state.
- `public/useLandingCameraFilters.js`: owns landing search, area, connection, ranking, and focus selection state.
- `public/useLandingPublicConfig.js`: hydrates public landing, ads, and Saweria settings.
- `public/useLandingReachability.js`: tracks public backend reachability.

## Critical Flow Rules

- Public popup URL sync must not override a newer local popup intent while React Router still exposes an older `camera` param.
- `useLandingInteractions` uses a pending URL camera id sentinel: camera switches wait for matching URL id, and close waits for no camera param before normal URL-driven selection resumes.
- Related camera switches and stream resolver replacements should replace history instead of pushing entries.

## Verification

- Public interaction changes: `npm test -- useLandingInteractions.test.js LandingPage.test.jsx VideoPopup.test.jsx`.
- Full frontend gate remains `npm test && npm run build && npm run lint`.
