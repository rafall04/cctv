# frontend/src/.module_map.md

## Entry Points

- `main.jsx`: React mount point.
- `App.jsx`: Provider tree, public/admin routes, lazy admin pages, public/admin playback scope wiring.
- `index.css`: Tailwind/global styles.
- `layouts/AdminLayout.jsx`: Active admin shell/navigation.
- `pages/`: Route-level orchestration. Keep large pages thin by extracting hooks, pure utils, and presentation components.
- `components/`: Shared UI, public landing components, admin components, playback components, video/multiview components.
- `contexts/`: Global theme, branding, security, notification, camera, timezone, and toast state.
- `hooks/`: Reusable page/data/effect hooks.
- `services/`: API clients and domain service wrappers.
- `utils/`: Pure helpers for URLs, delivery, playback selection, map coordinates, media, validation, and throttling.
- `test/` and `__tests__/`: Test setup/helpers.

## Route Ownership

- Public landing: `pages/LandingPage.jsx`, `components/landing/*` including `LandingPageSimple.jsx`, `hooks/public/*`, `components/MapView.jsx`, `components/MultiView/*`.
- Public playback: `pages/Playback.jsx` with `accessScope='public_preview'`, `hooks/playback/*` including `usePlaybackSelectionActions.js`, `utils/playbackUrlState.js`, `components/playback/*`, `services/recordingService.js`, `services/playbackViewerService.js`.
- Admin playback: same `Playback.jsx` with `accessScope='admin_full'` under `AdminPageRoute`.
- Dashboard: `pages/Dashboard.jsx`, `hooks/admin/useDashboardData.js`, dashboard widgets in `components/admin/dashboard/*` plus shared dashboard widgets in `components/*`.
- Cameras: `pages/CameraManagement.jsx`, `hooks/admin/useCameraManagementPage.js`, `components/admin/cameras/*`, `utils/admin/cameraFormAdapter.js`.
- Areas: `pages/AreaManagement.jsx`, `services/areaService.js`, `utils/areaCoverage.js`.
- Analytics: `pages/ViewerAnalytics.jsx`, `pages/PlaybackAnalytics.jsx`, `hooks/admin/useViewerAnalytics*`, `components/admin/analytics/*`.
- Recording: `pages/RecordingDashboard.jsx`, `hooks/admin/useRecordingDashboardData.js`, `components/admin/recordings/*`.
- Settings: `pages/UnifiedSettings.jsx`, `components/admin/settings/*` including legacy tab wrappers, settings/branding/saweria services; read `components/admin/settings/.module_map.md` before panel edits.
- Legacy settings/components: `components/settings/*` compatibility re-exports, `components/AdminLayout.jsx`, and `components/LandingPageSimple.jsx` compatibility re-export are not preferred for new work.

## Critical Flows

- App bootstrap: `main.jsx` -> `App.jsx` -> providers -> routes -> lazy pages.
- Public live camera: `LandingPage` -> camera context/config hooks -> landing filters/grid/map with reusable viewer stat badges -> `VideoPopup`/`VideoPlayer` with popup header viewer stats; proxied HLS view sessions are backend-owned, while direct/external non-proxy streams are frontend-owned.
- Map chrome overlays: `MapView.jsx` delegates top chrome to `components/maps/MapTopChrome.jsx`; decorative hints must keep pointer-event transparent so Leaflet controls such as Hybrid/Street layers remain clickable, and only concrete controls like the area filter panel should opt back into pointer events.
- Public playback: URL `cam`/`t` params -> `utils/playbackUrlState.js` -> `Playback.jsx` camera shell -> `hooks/playback/usePlaybackSegments.js` -> segment URL/media lifecycle hooks -> playback viewer session.
- Admin playback: protected route -> same playback page with full scope. Do not leak admin scope into public share links.
- Camera CRUD: page hook -> camera form components -> `cameraService` -> optimistic UI/cache refresh; Camera Management recording payloads should preserve `enable_recording` for recordable HLS delivery types (`internal_hls`, `external_hls`) and can set internal RTSP transport plus thumbnail strategy overrides.
- Area bulk policy: `AreaManagement` -> `areaService` -> preview/submit policy update -> refresh overview; area form owns internal ingest and RTSP transport defaults for cameras without overrides.
- Analytics: filter state hook -> admin services -> analytics components/tables.
- Runtime config: `ApiClientInitializer`/runtime config should load before API consumers depend on base URL.

## State And Hook Rules

- Keep all hooks before conditional returns.
- Use refs for async media/session state that event handlers read after state changes.
- Guard camera/segment playback transitions so stale selected segment cannot build a stream URL for a new camera.
- Clean up timers, intervals, event listeners, media listeners, and in-flight async effects.
- URL params: live uses `camera`; playback uses `cam` and `t`.

## Styling Rules

- Use Tailwind classes and existing UI primitives.
- Match admin pages to dense operational UI; avoid landing-page composition in admin tools.
- Keep cards for repeated items/modals/tools, not nested page sections.
- For icon buttons, prefer existing icon library/components.
- Verify responsive text containment when changing UI layout.

## Stabilization Targets

- `components/MapView.jsx`: top chrome is extracted to `components/maps/MapTopChrome.jsx`; continue extracting map state/effects, marker data shaping, and modal control into focused hooks/components.
- `pages/AreaManagement.jsx`: extract cards/forms/policy preview and page hook logic.
- `pages/Playback.jsx`: route shell should delegate URL state to `utils/playbackUrlState.js`, segment loading, viewer/media lifecycle, selection/reset actions, and share/snapshot behavior to `hooks/playback/*`; keep future playback features in these boundaries.
- `components/MultiView/VideoPopup.jsx` and `components/VideoPlayer.jsx`: extract media state machines and controls.
- `pages/Dashboard.jsx`: route shell now delegates stream panels, skeletons, summary cards, and sidebar to `components/admin/dashboard/*`; continue extracting route-local header/error helpers if dashboard grows again.
- `pages/UserManagement.jsx`, `pages/SponsorManagement.jsx`, settings panels: split form/state concerns before adding fields.
- Keep `package.json` full `src` lint passing as new pages, hooks, and utilities are added.

## Verification

- Full frontend gate: `npm test && npm run build && npm run lint`.
- Focused test: `npm test -- <test-file>`.
- For playback/media changes, run `npm test -- Playback.test.jsx PlaybackVideo.test.jsx`.
- For landing/map changes, run relevant `LandingPage`, `MapView`, landing component, and public hook tests.
- For admin page changes, run target page test plus service/hook tests if present.
