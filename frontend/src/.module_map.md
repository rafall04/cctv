# frontend/src/.module_map.md

## Entry Points

- `main.jsx`: React mount point.
- `App.jsx`: Provider tree, public/admin routes, lazy admin pages, public/admin playback scope wiring.
- `index.css`: Tailwind/global styles.
- `layouts/AdminLayout.jsx`: Active admin shell/navigation.
- `pages/`: Route-level orchestration. Keep large pages thin by extracting hooks, pure utils, and presentation components.
- `components/`: Shared UI, public landing components, admin components, playback components, video/multiview components.
- `contexts/`: Global theme, branding, security, notification, camera, timezone, and toast state.
- `hooks/`: Reusable page/data/effect hooks.
- `services/`: API clients and domain service wrappers.
- `utils/`: Pure helpers for URLs, delivery, playback selection, map coordinates, media, validation, and throttling.
- `test/` and `__tests__/`: Test setup/helpers.

## Route Ownership

- Public area pages: `pages/AreaPublicPage.jsx`, `utils/areaPublicRanking.js`, `services/publicGrowthService.js`, `utils/publicGrowthShare.js`, and `components/landing/LandingTrendingCameras.jsx`; all public area links should be built through `publicGrowthShare` so raw area names and canonical `area_slug` values resolve to the same `/area/:areaSlug` route.
- Public discovery/PWA: `components/landing/LandingDiscoveryStrip.jsx`, `components/landing/LandingSmartFeed.jsx`, `components/PwaInstallPrompt.jsx`, `utils/landingCameraInsights.js`, `utils/publicLandingSections.js`, `utils/sanitizePublicHtml.js`, `utils/preloadLandingMapView.js`, `services/publicGrowthService.js`, `services/publicCameraResolver.js`, `utils/registerServiceWorker.js`, `public/site.webmanifest`, and `public/sw.js`.

- Areas: `pages/AreaManagement.jsx`, `services/areaService.js`, `utils/areaCoverage.js`.

## Critical Flows

- App bootstrap: `main.jsx` -> `App.jsx` -> providers -> routes -> lazy pages.
- Public live camera: `LandingPage` -> `hooks/public/useLandingPageController.js` plus camera context/config hooks -> sanitized `/api/cameras/active` read model for landing/map lists -> landing filters/grid/map with precomputed search indexes, adaptive progressive render windows, and reusable viewer stat badges -> on-demand `/api/stream/:id` resolution when a camera opens; grid and map popups both resolve the camera payload before `VideoPopup`/`VideoPlayer` mounts so the player never receives metadata-only map rows -> shared media cleanup, popup header stats, camera detail sheet, and related cameras; proxied HLS view sessions are backend-owned, while direct/external non-proxy streams are frontend-owned.
- Public growth discovery: landing full/simple modes and `/area/:areaSlug` -> `publicGrowthService` -> sanitized `/api/public/*` area/trending/discovery endpoints plus local camera payload insight scoring -> `publicCameraResolver` when stream URLs are missing -> `LandingDiscoveryStrip`/`LandingSmartFeed`/`AreaPublicPage`/`LandingTrendingCameras` -> existing `VideoPopup` with minimal detail and related-camera handoff; simple mode uses the compact smart-feed variant with fewer sections/items, discovery strips and search dropdowns cap active items without sync effects, smart-feed sections memoize by camera/time inputs, camera cards prewarm the video popup chunk on first user intent, public camera refresh pauses while video/multiview surfaces are active, and grid/area pages progressively mount camera cards with stable card props so large public camera sets stay light on mobile.
- PWA mobile app: `main.jsx` calls `registerServiceWorker()` after runtime config loads; `App.jsx` renders `PwaInstallPrompt` only in the public route shell after browser install capability appears; `/site.webmanifest` defines install metadata and shortcuts, while `/sw.js` caches app-shell assets and skips API/HLS streams.
- Map chrome overlays: `MapView.jsx` delegates top chrome to `components/maps/MapTopChrome.jsx`; public landing preloads the map chunk from `utils/preloadLandingMapView.js` when non-low-end devices approach the camera workspace or when mobile users choose Map, while decorative hints must keep pointer-event transparent so Leaflet controls such as Hybrid/Street layers remain clickable, and only concrete controls like the area filter panel should opt back into pointer events.
- Public playback: canonical `/playback?cam=:cameraSlug&t=:timestamp` URLs and legacy root `/?view=playback&cam=:cameraSlug&t=:timestamp` URLs -> `utils/playbackUrlState.js` -> `Playback.jsx` camera shell -> `hooks/playback/usePlaybackSegments.js` -> segment URL/media lifecycle hooks -> playback viewer session; public playback keeps a lightweight quick-actions strip for live CCTV, area, and home routes without loading the full landing discovery shell; cameras with no recording segments must stay inside the playback shell with the camera selector visible, while true public access denial can use the dedicated denied state.
- Admin playback: protected route -> same playback page with full scope. Do not leak admin scope into public share links.
- Camera CRUD: page hook -> camera form components -> `cameraService` -> optimistic UI/cache refresh; Camera Management recording payloads should preserve `enable_recording` for recordable HLS delivery types (`internal_hls`, `external_hls`) and can set internal RTSP transport plus thumbnail strategy overrides.
- Area bulk policy: `AreaManagement` -> `areaService` -> preview/submit policy update -> refresh overview; area form owns internal ingest and RTSP transport defaults for cameras without overrides.
- Analytics: filter state hook -> admin services -> analytics components/tables.
- Runtime config: `ApiClientInitializer`/runtime config should load before API consumers depend on base URL; `TimezoneProvider` reads `/api/settings/timezone` publicly and must not depend on admin auth during public landing bootstrap.
- Timestamp display: use `TimezoneContext` helpers with explicit `TIMESTAMP_STORAGE` modes. Use `LOCAL_SQL` for live/playback viewer session history, `UTC_SQL` for SQLite `CURRENT_TIMESTAMP` metadata such as token audit/user/feedback rows, and default/ISO handling for recording segment timestamps.

## State And Hook Rules

- Keep all hooks before conditional returns.
- Use refs for async media/session state that event handlers read after state changes.
- Guard camera/segment playback transitions so stale selected segment cannot build a stream URL for a new camera.
- Clean up timers, intervals, event listeners, media listeners, and in-flight async effects.
- URL params: live uses `camera`; playback uses `cam` and `t`.

## Styling Rules

- Use Tailwind classes and existing UI primitives.
- Match admin pages to dense operational UI; avoid landing-page composition in admin tools.
- Keep cards for repeated items/modals/tools, not nested page sections.
- For icon buttons, prefer existing icon library/components.
- Verify responsive text containment when changing UI layout.

## Stabilization Targets

- `components/MapView.jsx`: top chrome is extracted to `components/maps/MapTopChrome.jsx`; continue extracting map state/effects, marker data shaping, and modal control into focused hooks/components.
- `pages/AreaManagement.jsx`: extract cards/forms/policy preview and page hook logic.
- `pages/Playback.jsx`: route shell should delegate URL state to `utils/playbackUrlState.js`, segment loading, viewer/media lifecycle, selection/reset actions, and share/snapshot behavior to `hooks/playback/*`; keep future playback features in these boundaries.
- `components/MultiView/VideoPopup.jsx` and `components/VideoPlayer.jsx`: extract media state machines and controls.
- `pages/Dashboard.jsx`: route shell now delegates stream panels, skeletons, summary cards, and sidebar to `components/admin/dashboard/*`; continue extracting route-local header/error helpers if dashboard grows again.
- `pages/UserManagement.jsx`, `pages/SponsorManagement.jsx`, settings panels: split form/state concerns before adding fields.
- Keep `package.json` full `src` lint passing as new pages, hooks, and utilities are added.

- For landing/map changes, run relevant `LandingPage`, `MapView`, landing component, and public hook tests.
- For admin page changes, run target page test plus service/hook tests if present.
