# frontend/src/pages/.module_map.md

## Public Pages

- `LandingPage.jsx`: public live CCTV experience, landing layout modes, map/grid/playback entry points, public config, and ads/feedback placement.
- `LoginPage.jsx`: admin login, password expiry warning, auth redirect.
- `Playback.jsx`: shared public/admin playback route shell. Public scope is `public_preview`; admin scope is `admin_full`; URL state, segment loading, and media/viewer lifecycle live in `../utils/playbackUrlState.js` and `../hooks/playback/*`.

## Admin Pages

- `Dashboard.jsx`: admin overview route shell; delegates dashboard skeletons, stream panels, summary cards, and sidebar widgets to `../components/admin/dashboard/*`.
- `CameraManagement.jsx`: camera CRUD shell using `hooks/admin/useCameraManagementPage.js` and `components/admin/cameras/*`.
- `AreaManagement.jsx`: area CRUD, area overview, default grid settings, bulk camera policy preview/apply.
- `UserManagement.jsx`: admin user CRUD and validation.
- `FeedbackManagement.jsx`: feedback listing/detail/filter management.
- `ViewerAnalytics.jsx`: live viewer analytics shell using `components/admin/analytics/*`.
- `PlaybackAnalytics.jsx`: playback viewer analytics shell.
- `RecordingDashboard.jsx`: recording overview, camera recording controls, restart logs.
- `UnifiedSettings.jsx`: active settings hub using `components/admin/settings/*`.
- `SponsorManagement.jsx`, `SaweriaSettings.jsx`, `BrandingSettings.jsx`, `Settings.jsx`: sponsor/donation/legacy settings surfaces.
- `admin/HealthDebug.jsx`, `admin/ImportExport.jsx`, `admin/BackupRestore.jsx`: admin tools pages.

## Shared Page Flows

- Route mounting is controlled by `../App.jsx`; protected pages render inside `layouts/AdminLayout.jsx`.
- Public/admin playback must keep access scope separate and must not leak admin scope into public share URLs; use `../utils/playbackUrlState.js` for playback params.
- Public popup camera clicks should set the selected camera immediately with `_stream_resolution_pending: true`, then replace it with the resolved `/api/stream/:id` payload and `_stream_resolution_pending: false`; guard pending resolver promises so stale results cannot reopen or overwrite a newer popup.
- Landing URL sync must not re-select a raw camera when the current popup already has the same camera id; otherwise it strips pending stream metadata and causes premature popup error flashes.
- Popup URL history policy: initial page-to-popup opens may push a `camera` entry, but related-camera switches, resolved stream replacements, and popup close must use history replace so users do not need to close/back through previous popup cameras.
- Popup URL sync is hook-owned in `../hooks/public/useLandingInteractions.js`; local popup open/close intent must suppress stale router params until the URL catches up.
- URL params: live public uses `camera`; playback uses `cam` and `t`.
- Page data should move into `../hooks/admin/*` or `../hooks/public/*` before page files grow further.
- Presentation should move into `../components/*`; pure selection/formatting should move into `../utils/*`.

## High-Risk Pages

- `AreaManagement.jsx`: combines page data, card UI, policy preview, forms, and bulk operations.
- `Playback.jsx`: combines URL sync, segment loading, media source lifecycle, tracking sessions, snapshot/share, ads, and UI state.
- `Dashboard.jsx`: dashboard data orchestration is separated from most widget rendering; remaining route-local header/error helpers should be extracted if the shell grows again.
- `UserManagement.jsx` and `SponsorManagement.jsx`: CRUD/form state should be split before adding new fields.
- `LoginPage.jsx`: auth/session behavior and timers should remain focused and well tested.

## Extraction Targets

- `Playback.jsx`:
  - any future playback feature state that does not belong in `../hooks/playback/*`
- `AreaManagement.jsx`:
  - area card/list components
  - bulk policy preview component
  - form modal/state hook
- `Dashboard.jsx`:
  - optional extraction of route-local header and error state helpers if the page grows again.
- `LandingPage.jsx`:
  - keep public mode state in hooks and landing components; do not add map/video logic inline.
